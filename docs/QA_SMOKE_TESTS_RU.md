# QA Smoke Tests

## Цель

Проверить, что ключевые пользовательские сценарии работают после изменения кода.

## Подготовка

1. Открыть Excel и подключить `AI_Assistant.xlam`.
2. Открыть тестовую книгу с 2-3 листами и данными.
3. Убедиться, что настроен хотя бы один провайдер (DeepSeek/OpenRouter/LM Studio).

## Smoke-сценарии

### 1. Базовый чат без выполнения команд

- Шаги:
  1. Открыть окно ассистента.
  2. Отправить запрос без `commands` (например: "Опиши структуру данных").
- Ожидание:
  - Ответ отображается в чате.
  - Ошибок VBA нет.

### 2. Простейшая команда изменения данных

- Шаги:
  1. Попросить ассистента установить значение в одну ячейку.
- Ожидание:
  - Ячейка изменена.
  - В чате есть отметка о выполнении команд.

### 3. Команда форматирования

- Шаги:
  1. Попросить сделать `BOLD` и `FILL_COLOR` для диапазона.
- Ожидание:
  - Формат применён корректно.

### 4. Команда с графиком

- Шаги:
  1. Попросить создать график `CREATE_CHART`.
- Ожидание:
  - График создан на активном листе.

### 5. Локальный режим LM Studio

- Шаги:
  1. Включить локальный режим в настройках.
  2. Отправить короткий запрос.
- Ожидание:
  - Ответ получен через локальную модель.

### 6. Vision-сценарий

- Шаги:
  1. Прикрепить изображение в чате.
  2. Попросить извлечь данные.
- Ожидание:
  - Модель обрабатывает изображение и возвращает осмысленный результат.

## Критерий прохода

- Все 6 сценариев проходят без runtime-ошибок VBA.
- Нет зависаний UI формы.
- Нет неожиданных изменений на других листах.
